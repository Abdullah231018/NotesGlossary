Data representation

Binary systems

The binary system is based on the number 2. It is made up of 1s and 0s.

Converting Binary to Denary

To calculate a binary number you will have to place it in columns of base 2 numbers. You will be required to add all the base 2 numbers to finish the conversion.
For example: Let's convert 11010100 to Denary.

128 64 32 16 8 4 2 1
1 1 0 1 0 1 0 0

128 + 64 + 16 + 4 = 212

Converting Denary to Binary

Now we will look at how to do the opposite. To calculate denary to binary we will have to set up the columns of base 2 numbers again. We are required to get the Denary value from the column of base 2 numbers.
For example: Let's convert 150 to Binary

128 64 32 16 8 4 2 1
1 0 0 1 0 1 1 0

To get 150 we will require a 128, then we are left with 22. hence, we will skip 64 and 32 as they are larger values. We will use 16 and we have a remaining of 6. Finally, a 4 and 2 will give us the 6 we need.

128 + 16 + 4 + 2 = 150

Hexadecimal

It is a base 16 system. Number 0 - 9 and letters A - F are used to represent all the hexadecimal digits. 0 - 9 each have denary value of 0 - 9. A - F are the rest of the 6 digits from 10 - 15.

Converting Binary to Hexadecimal

To covnert Binary to Hexadecimal we will have to create two 4 bits.
For example: 11000101

8 4 2 1 - 8 4 2 1
1 1 0 0 - 0 1 0 1

The first section we get 12 which when converted to hexadecimal we get C. The second half we get 5. Hence, our hexadecimal value will be C5

Converting Hexadecimal to Denary

To do this we will have to convert hexadecimal to binary first and then we convert our result to binary.
For example: we will convert 2B to Denary. First we convert Hex to Binary.
First step is to split the Hex (2 - B) = (2 - 11)

8 4 2 1 - 8 4 2 1
0 0 1 0 - 1 0 1 1

The binary we get is 00101011. Now we will have to convert our Binary to Denary. We go back to using the 8 bit

128 64 32 16 8 4 2 1
0 0 1 0 1 0 1 1

32 + 8 + 2 + 1 = 43

Error checking

There are four commons methods for error checking.

Parity checking

Parity checking is used to chefck whether data has been changed or corrupted following transmission from one device to another.

Automatic repeat request (ARQ)

ARQ uses acknowledgements and timeouts to achieve reliable data transmission.

Checksum

A checksum is a small sized block of data derived from another block of digital data for the purpose of detecting errors.

Echo check

Echo check determines the datas integrity by returning the data to the source and comparing it. It is a very unrealiable method.

Lossless and Lossy File Compression

Lossless file compression

This method compressses the file by reconstructing data and when uncompressed it does not lose any data.
Examples of lossless compression: RAW, GZIP, BMP and PNG

Lossy file compression

This method compresses the file by eliminating unnecessary bits of data. It is not posible to retain the original file.
Examples of lossy compression: JPEG and MP3.
